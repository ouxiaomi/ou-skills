# Git 提交消息参考文档

## 常见提交前缀风格

| 前缀 | 用途 | 示例 |
|------|------|------|
| `feat:` | 新功能 | `feat: 添加用户登录功能` |
| `fix:` | 修复 bug | `fix: 修复首页加载失败问题` |
| `doc:` | 文档更新 | `doc: 更新 README 安装说明` |
| `docx:` | 文档更新（变体） | `docx: 补充 API 使用示例` |
| `refactor:` | 代码重构 | `refactor: 重构配置加载逻辑` |
| `style:` | 代码格式调整 | `style: 统一代码缩进` |
| `test:` | 测试相关 | `test: 添加单元测试覆盖` |
| `chore:` | 构建过程或工具变动 | `chore: 升级依赖包版本` |
| `perf:` | 性能优化 | `perf: 优化数据库查询性能` |
| `ci:` | CI 配置更新 | `ci: 添加自动化测试流程` |

## 提交消息最佳实践

1. **简洁明了**：使用祈使句，一般现在时
2. **单行原则**：第一条消息不超过 50 个字符
3. **关注"是什么"而非"为什么"**：详细说明在 body 部分
4. **遵循仓库风格**：查看最近提交，模仿其前缀和语言

## Git 命令速查

```bash
# 查看暂存状态
git status --short

# 查看暂存区变更
git diff --cached

# 查看暂存区统计
git diff --cached --stat

# 查看最近 5 条提交消息
git log -5 --pretty=format:%s

# 查看最近 5 条提交详细信息
git log -5 --pretty=format:"%h - %an, %ar : %s"
```
